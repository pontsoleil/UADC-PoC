# 決定:現在の統合文書を5つのガイドに

## コンテンツ

トピック固有のサブディレクトリの重複したセットアップ、操作、テスト、および内部処理の説明。 リーダーは、どのファイルが現在の仕様を含んでいるかを簡単に伝えることができませんでした。

## デコレーション

1. 現在の 文書 を 5 つのチャプター番号のファイルで保持します。
**docs/**:**SETUP.md**、**TUTORIAL.md**、**SYNTAX_BINDING.md**、**SEMANTIC_BINDING.md**、および**DATA_MODEL.md**。
2. 操作手順と機能レベルの実装の詳細を含める
別のスクリプト処理マニュアルを維持する代わりに関連するガイド。
3. **docs/decisions/**を歴史的建造物として保持します。
4. 各スクリプト指向のディレクトリに、短い**README.md**を配置します。
**docs/**に初めてナビゲートすることなくコンテンツを理解することができます。
5. Markdownをソースとして扱い、PDFを同期リリース用に再生します。

## コンシーケンス

- 5つのガイドは、現在の規範的な人間が読める唯一の文書です。
- 古い重複したガイドディレクトリは、独自のコンテンツが削除された後に削除されます
移行。
- スクリプトの動作は、該当するガイドとディレクトリのREADMEを更新します。
同じ変更。
- アーキテクチャレコードは簡潔であり、現在のガイドを参照します。
実装の詳細を復元します。
