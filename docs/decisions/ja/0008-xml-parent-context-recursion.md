# 決定: Phase 1 の XML 親コンテキスト再帰を使用する フォワード変換

## コンテンツ

UADC Structured CSV は**dInvoice**や**dInvoiceLine**などの ディメンション列 で繰り返されたセマンティッククラスを表しています。 ネストされた 反復クラス は、正しい親の XML コンテキスト内で、子 ディメンション が正しい親の発生に属しているように解釈しなければなりません。

## デコレーション

1. **BindingClass**ツリーを構文バインディング**C**行からビルドします。
2. そのクラスツリーを繰り返し歩くことでUBL XMLを処理します。
3. 各クラスでは、直接**A**行を現在のXMLに相対的に評価します。
コンテキスト。
4. 非反復子クラスの現在のStructured CSV行を再利用します。
5. 反復子クラス の場合、XML の子コンテキストを現在の内部に列挙します。
親のコンテキスト。
6. 繰り返された ディメンション 値を 1 ベースの発生数として割り当てる
現在の親コンテキスト。
7. ネストされた繰り返し行を作成する際の ancestor ディメンション 値を保存します。

## コンシーケンス

- **dInvoiceLine**値が インボイス で指定されます。
インボイス.
- **dItemAttributes**などのネストされた ディメンション は、現在
インボイス 行、ドキュメント全体でグローバルにはありません。
- ネストされたリピートのための行は、このような親ディメンションを、運びます
**dInvoice=1**,**dInvoiceLine=2**,**dItemAttributes=1**.
- 前方 Phase 1 コンバーターは バインディング表 によって使用される同じ親子のコンテキストモデルに従います。
- 逆変換はバインディング表パスとUBLの子注文ルールを使用してXMLを構築します。
