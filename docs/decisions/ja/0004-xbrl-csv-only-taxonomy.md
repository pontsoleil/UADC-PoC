# 決定: 生成 xBRL-CSV タクソノミ 詳しくはこちら

## コンテンツ

xBRL-CSV タクソノミ 出力から出力されたXBRL 2.1のタプル指向の タクソノミ を区別する初期実験。 UADC PoC 構造 CSV は、 xBRL-CSV として検証されるように意図されているため、追加の概念とタプルエントリーポイントスキーマは、現在のチェックポイントをサポートせずに複雑さを追加します。

## デコレーション

1. **en16931-oim-<version>.xsd**xBRL-CSV スキーマを生成します。
2. **en16931-oim**でハイパーキューブ、ディメンション、およびプライマリアイテムの概念を生成します。
3. **en16931-content-<version>.xsd**を生成しないでください。項目宣言は、モジュールスキーマとxBRL-CSV の主な項目は**en16931-oim**に残ります。
4. リファレンス**en16931-def-<version>.xml**, ラベル, EN 16931 表示リンクベース エントリーポイント. プレゼンテーション Class/BG ノードは OIM の主な項目と BT ノードがモジュールの事実に解決します。
5. **plt-all-<version>.xsd**を生成しないでください。
6. このPoCのXBRL 2.1タプル**complexType**構造を定義しないでください。
7. **../gen/gl-gen-<version>.xsd**として生成されたモジュールスキーマから参照**gl-gen**。

## コンシーケンス

- タクソノミ 出力は、構造化された CSV 検証対象と一致します。
- Arelle 検証は xBRL-CSV DTS エントリ ポイントに焦点を当てています。
- Tuple 固有の実装とテストは、現在の 適用範囲 から削除されます。
- 将来のタプル タクソノミ は、必要に応じて、別の設計として処理する必要があります
