# 階層 CSV の結合のSpec

このPoCは、構文結合出力用のディメンションベースの階層CSVを使用します。

## コラムグループ

列は2つのグループに分割されます。

1. BG/class ディメンション
   - LHM**type=C**行から定義します。
   - カラム名は**d**+ LHM**element**値の上限 キャメルケースです。
   - これらの列はCSVの先頭に整列されます。
   - **インボイス**は**dInvoice**で表されます。
   - **0.1**または**1..1**multiplicityでBGはディメンション列を定義しません。
   - **インボイス 行**のような BG を繰り返して ディメンション列 を定義します。

2. BT/fact 値
   - LHM**type=A**行から定義されます。
   - カラム名はLHM**element**の値です。
   - これらの列は、すべてのBG ディメンション列sに従う。

## 列のルール

各出力行は1つのBGの発生かインボイスの根を表します。

- インボイス ルート行に**dInvoice=1**と root レベルの BT 値があります。
- BG行を繰り返すと**dInvoice=1**と独自のディメンション値があります。
- **0..1**または**1.1**BGは別のディメンション行を作成しません。 BT値は、最も近い繰り返しの祖先行に書かれている、または**dInvoice=1**行に繰り返すのがない場合。
  - 売り手 および 買い手 郵便番号は**dInvoice=1**行に書かれています。
  - 最初の行の拡張子行は**dInvoice=1**、**dInvoiceLine=1**、および**インボイスLineNetAmount=1230**を持っています。
  - 2行目の線の伸縮量行は**dInvoice=1**、**dInvoiceLine=2**、および**インボイスLineNetAmount=560**を持っています。
- 繰り返し BG は、XML の発生順序で独自の ディメンション 値を増加させます。
- BT 値は、独自の BG の行に書かれています。

### 親子列分離

**dBbb**は**dAaa**の子で、その列は次のとおりです。

```csv
dAaa,dBbb,a1,a2,b1,b2,b3
```

**dBbb**が非対応の場合、 ディメンション 値または別の行はありません。 その事実は**dAaa**行に統合されます。

```csv
1,,a1V1,a2V1,b1V1,b2V1,b3V1
```

**dBbb**が繰り返されると、最初の発生を含むすべての発生は、別の子行に書かれています。 親行は**dAaa**の事実のみを含み、その親行の**dBbb**の事実は空です。

```csv
1,,a1V1,a2V1,,,
1,1,,,b1V1,b2V1,b3V1
1,2,,,b1V2,b2V2,b3V2
```

次の混合parent/child行は無効です。

```csv
1,1,a1V1,a2V1,b1V1,b2V1,b3V1
1,2,,,b1V2,b2V2,b3V2
```

逆変換は、この行スコープルールを検証し、ancestor ディメンションが所有する事実が繰り返された子行にポップアップしたときに行を拒否します。

## 拘束ルール

構文バインディング CSVは**semantic_path**と**xpath**を提供します。

- LHM 意味パス の場合、親 意味パス は、独自の BG を識別します。
- LHM**type=C**行でBGを独自に解決します。
- BGのXMLコンテキストはLHMクラス**xpath**です。
- **const:**結合行は、XML 生成値を固定し、XML-to-CSV 出力には無視されます。
- BT がバインディングで定義されていない場合、CSV カラムは空にします。

## バインディング表 権限

構文バインディングテーブルは変換の権限です。 必要なモデル列、クラス列、ファクト列、意味パスs、構造化されたCSVカラム名、およびXPath式を埋め込む。 コンバーターはバインディング表の列の順序をStructured CSVの出力特性保つのに使用します。

コンバーターは、結合行とソースXMLから解決できる値を書きます。 特定のサンプル文書がそれらの値を持っていない場合でも、バインディングモデルの一部である列は、レビューのために残っています。
